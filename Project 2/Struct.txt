        > -writesize: unsigned char

        > -write_busy: volatile unsigned char

    Member Functions :
        > +readbuf(addr:unsigned int, buf: unsigned char*, size: unsigned char) - reads a specified amount of data (size) from the EEPROM starting at a specified address (addr) and places it in the specified buffer (buf). 
            > read contiguous characters from EEPROM starting at addr and place them in buf[0:size-1]
        
        > +writebuf(addr:unsigned int, buf: unsigned char *, size: unsigned char) - places the data (specified by buf and size) into the write buffer for later writing to the EEPROM.  The addr parameter specifies the location to write the data to.  This function should not be called when another write is in progress.
            > 1. writeaddr = addr
            > 2. write_busy = 1
            > 3. bufidx = 0
            > 4. Copy buf[0:size-1]
            > 5. writesize = size
            > 6. Enable EEPROM ready interrupts

        > +isbusy():int - returns 0 if write_busy is 0, otheriwise, returns 1
            > return write_busy==1
        
        > -__vector_22() - Enabled when writebuf() places new data in the write buffer, this ISR sends one byte at a time to the EEPROM.  When the last byte is sent, it disables further EEPROM interrupts
            > if (bufidx<writesize)
                > 1. Write writebuf[bufidx] to writeaddr in the EEPROM
                > 2. writeaddr++
                > 3. bufidx++
            > if (bufidx>=writesize)
                > Disable EEPROM ready interrupts
                > write_busy = 0

VPD class :
    Internal State :
        > +vpd: vpd_struct - the vital product data read from the Eeprom at address 0x000. This data is represented as a structure and individual members can be accessed using dot notation (e.g. vpd.serial_number)
        > -defaults: vpd_struct - the factory defaults for the vpd
            > Use structure initialization syntax to set the following values for defaults. The VPD structure within the EEPROM should begin at address 0x0000
                - 'Field' : 'Value' : 'Description'
                - .token : "SER" : "N/A"
                - .model : "FirstName" : "must be <12 char"
                - .manufacturer : "LastName" : "must be <12 char"
                - .serial_number : "AnyString" : "must be <12 char"
                - .manufacture_date : 0 : "N/A"
                - .mac_address : {‘x1’,’x2’,’x3’,’y1’,’y2’,’y3} : x1 = ascii code for the first letter in your name anded with 0xFE asci codes for the second and third letters of your first name. y1-y3 = asci codes for the first three letters of your last name
                - .country_of_origin : "USA" : "N/A"
                - .checksum : 0 : "N/A"

    Member Functions : 
        > +init() - initializes vpd member data from the EEPROM.  If vpd data is invalid after initialization, the EEPROM is written to “factory defaults”, and the vpd data is reinitialized from the new EEPROM values

            > if (is_data_valid()) skip 2 & 3
            > 2. write defaults()
            > 3. initialize vpd data w/ eeprom.readbuf()

        > -is_data_valid(): int - returns 1 (true) if the vpd token is “SER” and the checksum is valid.  Otherwise, returns 0

        > -write_defaults() - called by init() to write the “factory defaults” to the EEPROM
            > update_checksum() for defaults

CONFIG class : 
    Internal State : 
        > +config: config_struct - the configuration data read from the Eeprom at address 0x040.  This data is represented as a structure and individual members can be accessed using dot notation (e.g. config.static_ip).  The data also acts as a write cache.  Data may be read/written while EEPROM updates are in progress
        > -modified: unsigned char - set by the set_modified() function, this variable signals that the data has been modified and should be written to eeprom during the next update cycle
        > -defaults: vpd_struct - the factory defaults for config
            > Use structure initialization syntax to set the following values for defaults. The Config structure within the EEPROM should begin at address 0x0040
                - 'Field' : 'Value'
                - .token : "ASU"
                - .hi_alarm : 0x3FF
                - .hi_warn : 0x3FE
                - .lo_alarm : 0x0000
                - .lo_warn : 0x0001
                - .use_static_ip : 0
                - .static_ip : {192,168,1,100}
                - .checksum : 0

    Member Functions : 
        > init() – initializes config member data from the EEPROM.  If config data is invalid after initialization, the EEPROM is written to “factory defaults”, and the config data is reinitialized from the new EEPROM values
            > if (eeprom.isbusy())
                > loop back to init() (indefinite loop till isbusy unlocks)
            > initialize config w/ eeprom.readbuf()
                > if (is_data_valid())
                    > skip to the end and returns
                > write_defaults()
                > initialize config w/ eeprom.readbuf()
                > clear the modified flag
                > return

        > update() – if config has been modified and eeprom is not busy, write all of the configuration data to the eeprom write buffer
            > if (eeprom.isbusy())
                > skip to the end & return 
            > if !modified
                > skip to the end & return
            > Update checksum
            > write config to eeprom
            > clear the modified flag
        
        > set_modified() – set the modified flag.  This function should be called any time the config data is modified
            > modified = 1
        
        > is_data_valid() – returns 1 (true) if the token is “ASU” and the checksum is valid. Otherwise, returns 0
            > result = 1
            > if (config.token != "ASU") 
                > result = 0
                    > return result
            > if (Checksum not valid)
                > result = 0
                    > return result
            > return result
        
        > write_defaults() – called by init() to write the config “factory defaults” to eeprom
            > update_checksum() for defaults
            > write defaults to eeprom

            • Parameters:
                • data – a pointer to the data structure to update the checksum
                • size – the size of the datastructure (use sizeof() to pass this parameter)
            • Description
                • This function changes the last byte of the data structure so that the sum of all the bytes within the data structure will be zero.

        > is_checksum_valid(char*data, unsigned int size):int 
            • Parameters:
                • data – a pointer to the data structure to validate
                • size – the size of the datastructure (use sizeof() to pass this parameter) 
            • Return:
                • 1 if the checksum is valid, 0 otherwise.
            • Description
                • This function sums all the byte values within the structure and returns 1 if the sum is zero.
